## A) 확정 요구사항 (Must / Should / Could)

### Must

- 멀티테넌트: 최고관리자(유니그린) + 고객사(테넌트) 데이터 분리
- 역할 최소셋: 고객사 대표(CEO) + 현장소장(모바일)
- 대표가 프로젝트 단위로 소장 가시성(보이기/숨기기) 설정
- 표준 업무 흐름: 방문/사진 → AI 진단 → 견적 → 계약 → 착공계 → 작업일지/사진 → 준공(준공계/대금청구/사진첩) → (일부) 수도광열비 서류 추적 → 하자/보증/세금계산서 → 입금
- 문서 출력 형식: 견적 Excel, 계약 PDF, 사진첩 PDF/DOCX, 시방서 PDF
- 전자서명: 공사도급계약 모두싸인(Modusign), 일용직 근로계약 자체 서명(서명이미지+타임스탬프)
- 세금계산서: 팝빌(Popbill) 발행/상태 추적
- 단가표: 최고관리자 PDF 업로드(연 2회) → 파싱/검수 → 업로드 시점부터 적용(버전)
- 앱(Android 우선) 범위: 사진 촬영/업로드, 진단/견적 조회, 신분증 촬영/업로드, 알림
- 일용직 근로자도 앱 사용 가능

### Should

- 과금: 아이디(좌석)당, 월/연, 1개월 트라이얼, PG 토스페이먼츠
- 가입/등록: 고객사/협력사/일용직 서류 기반 온보딩 + 사업자번호 검증/여성기업 여부 확인 흐름
- 노무: 지급명세서 생성 → 푸시 전달 → 수령확인, 월/분기 신고 엑셀 생성

### Could

- 나라장터 계정 연동 고도화
- 여성기업 여부 자동확인, HWP 자동생성, 발주처 포털 고도화

## B) 정보구조(IA) + 페이지 목록 (요약)

### Admin Web (최고관리자)

- 로그인
- 테넌트 관리(등록/상태/좌석/사용자)
- 단가표 업로드/버전 관리 + 스테이징 검수
- 외부 연동 상태(모두싸인/팝빌/토스)

### Tenant Web (고객사 대표)

- 대시보드(서명대기/발행대기/준공누락 등 요약)
- 프로젝트 허브(견적/계약/착공/작업/준공/하자/세금계산서/노무 탭)
- 사용자/권한 + 프로젝트 가시성(대표→소장)
- 고객/협력사 마스터(첨부 포함)
- 결제/구독(토스)

### Mobile (현장소장)

- 배정 프로젝트 목록/상세
- 사진 촬영/업로드(전/중/후/상세)
- 작업일지
- 일용직 등록(신분증/안전교육증/계좌/주소)
- 알림

### Mobile (일용직 근로자)

- 진입/인증(초대 링크/전화번호 인증)
- 내 근로계약(확인/서명)
- 지급명세서함(수령확인)
- 내 정보/서류(신분증/안전교육증/계좌/주소)

## C) 핵심 사용자 여정

- 견적(Excel) → 계약(모두싸인) → 착공 → 작업일지 → 준공서류 → 세금계산서(팝빌)/하자
- 현장(모바일) 사진/일지 축적 → 준공사진첩(PDF/DOCX) 생성
- 일용직 등록 → 근로계약(자체서명) → 지급명세서 → 수령확인 → 신고엑셀
- 단가표 PDF 업로드 → 파싱/검수 → 활성화 적용
- 구독/좌석 과금(토스) 트라이얼 → 유료전환 → 좌석 변경

## D) 목업 제작 계획 (2~3주 단위)

### Milestone 1 (0~2주)

- 프로젝트 1건이 견적→계약→착공→일지→준공→발행 상태까지 UI로 완주
- 문서 생성/다운로드 상태(queued/processing/done/failed) 포함

### Milestone 2 (3~5주)

- 노무(일용직/지급명세서/수령확인/신고엑셀)
- 마스터데이터(협력사/첨부)

### Milestone 3 (6~8주)

- 최고관리자(테넌트/단가표)
- 결제/세금계산서 상태머신 UX 확정

## E) 리스크/질문 (기본 가정 포함)

- 소장에게 숨길 정보 범위: 기본은 작업/사진/일지 중심, 금액/결제/세금 탭 숨김
- 모두싸인 실패/거부 시 대체: 재요청 + 오프라인 서명 PDF 업로드
- 팝빌/토스 상태코드 노출: 사용자용은 3~4단계 단순화, 상세는 관리자만
- 트라이얼 만료 제한: 읽기 가능 + 신규 생성/발행/연동 차단(업셀)
- 주민번호/신분증 저장 정책: 원본 최소 저장 전제 UX(마스킹/분리입력/권한 제한) 우선
