# 온보딩 문구 및 권한별 사용자 플로우/화면 정의

## 목적

회원 등록부터 실제 사용 시작까지의 단계별 안내 문구와 화면 정의를 역할별로 고정합니다.

---

## 공통 원칙

- 사용자는 최소 입력으로 시작하고, 필수 서류는 이후 단계에서 안내
- 40~50대 사용자 고려: 간결한 문장, 큰 버튼, 한 화면 1~2개의 행동
- 단계별 상태: 완료/대기/필요/오류
- 문서/서명/정산 등 중요 작업은 “왜 필요한지” 짧게 설명

---

## A. 고객사 대표(CEO) - Web 온보딩

### A-1. 회원가입 시작 (웹)

**페이지**: `/signup`
**목적**: 회사 계정 생성
**필드**: 회사명, 사업자등록번호, 대표자 이름/연락처/이메일, 비밀번호
**CTA**: “회사 등록하고 시작하기”
**안내 문구**:

- “사업자 정보를 먼저 등록하면 프로젝트 관리가 바로 시작됩니다.”

### A-2. 사업자등록증 업로드

**페이지**: `/signup/business-license`
**목적**: 회사 검증 자료 확보
**필드**: 사업자등록증 파일
**CTA**: “업로드 후 다음”
**안내 문구**:

- “사업자등록번호 검증에 사용됩니다. 제출 후에도 수정 가능합니다.”

### A-3. 담당자/소장 등록

**페이지**: `/onboarding/invite`
**목적**: 현장 사용자 초대
**필드**: 담당자(최대 2명) 이름/이메일/연락처, 소장 계정(이메일/연락처)
**CTA**: “초대 보내기”, “나중에 하기”
**안내 문구**:

- “현장 소장은 모바일 앱으로 작업일지/사진을 등록합니다.”

### A-4. 프로젝트 생성

**페이지**: `/projects/new`
**목적**: 첫 프로젝트 생성
**필드**: 프로젝트명, 주소, 고객명/연락처(선택), 메모
**CTA**: “프로젝트 만들기”
**안내 문구**:

- “프로젝트가 생성되면 견적→계약→준공까지 한 화면에서 관리됩니다.”

### A-5. 프로젝트 가시성 설정

**페이지**: `/access/project-visibility`
**목적**: 소장에게 보일 프로젝트 지정
**필드**: 프로젝트 목록 + 토글
**CTA**: “저장”
**안내 문구**:

- “소장은 선택된 프로젝트만 볼 수 있습니다.”

## B. 현장소장 - Mobile 온보딩

### B-1. 로그인

**페이지**: `/login` (모바일 앱)
**목적**: 계정 로그인
**필드**: 이메일/비밀번호
**CTA**: “로그인”
**안내 문구**:

- “대표가 초대한 계정으로 로그인하세요.”

### B-2. 배정 프로젝트 확인

**페이지**: `/projects`
**목적**: 담당 프로젝트 확인
**행동**: 프로젝트 카드 클릭
**안내 문구**:

- “현장에서 작업할 프로젝트만 표시됩니다.”

### B-3. 작업일지/사진 등록

**페이지**: `/projects/[id]` → `/projects/[id]/visits/new`
**목적**: 방문기록/사진 업로드
**필드**: 방문 유형, 시간, 메모, 사진(전/중/후)
**CTA**: “저장”
**안내 문구**:

- “카카오톡 대신 앱에 사진과 작업 내용을 남겨 주세요.”

### B-4. 일용직 등록

**페이지**: `/workers/new`
**목적**: 현장 근로자 등록
**필드**: 신분증/교육이수증 촬영, 계좌, 주소, 실거주 확인
**CTA**: “등록하기”
**안내 문구**:

- “주민등록번호 뒷자리는 마스킹되며, 법령에 따라 관리됩니다.”

## C. 일용직 근로자 - Mobile (근로자 모드)

### C-1. 간편 접속(인증)

**페이지**: `/worker/entry`
**목적**: 초대 링크 또는 전화번호 인증
**필드**: 휴대폰 번호
**CTA**: “인증번호 받기”
**안내 문구**:

- “초대받은 링크 또는 휴대폰 인증으로 바로 시작합니다.”

### C-2. 근로계약서 확인/서명

**페이지**: `/worker/contracts/[id]`
**목적**: 계약 내용 확인 및 서명
**필드**: 현장명, 일급, 근로일자, 직종, 계약서 전문
**CTA**: “서명하기”
**안내 문구**:

- “서명은 이미지로 저장되고 시간이 기록됩니다.”

### C-3. 지급명세서 확인/수령

**페이지**: `/worker/paystubs` → `/worker/paystubs/[id]`
**목적**: 급여 지급 확인 및 수령 처리
**CTA**: “수령 확인”
**안내 문구**:

- “수령 확인을 누르면 확인 기록이 남습니다.”

### C-4. 본인 서류 제출

**페이지**: `/worker/profile`
**목적**: 신분증/안전교육증/계좌 등 제출 상태 확인
**CTA**: “제출/수정”
**안내 문구**:

- “필수 서류가 제출되어야 지급 및 계약이 원활합니다.”

### D-1. 테넌트 생성

**페이지**: `/sa/tenants`
**목적**: 고객사 신규 등록
**필드**: 회사명, 사업자번호, 대표자 정보, 좌석 수, 트라이얼 기간
**CTA**: “테넌트 생성”
**안내 문구**:

- “생성 후 고객사가 바로 로그인 가능합니다.”

### D-2. 단가표 업로드/검수

**페이지**: `/pricebooks` → `/pricebooks/revisions/[id]/staging`
**목적**: 단가 업데이트
**필드**: PDF 업로드, 스테이징 검수
**CTA**: “업로드”, “활성화”, “정식 반영”
**안내 문구**:

- “업로드 시점부터 새 단가가 적용됩니다.”

## 권한별 화면 정의 요약

### Web (Admin)

- 최고관리자: 테넌트 관리, 단가표 업로드/검수, 연동 상태
- CEO: 프로젝트 전 과정, 견적/계약/준공/세금계산서, 노무, 사용자/권한, 결제/구독

### Mobile

- 현장소장: 배정 프로젝트, 방문기록/사진, 작업일지, 일용직 등록
- 근로자: 인증, 근로계약 서명, 지급명세서 수령확인, 서류 제출
