## 1. 프로젝트 개요

### 1.1 서비스 소개
- **서비스명**: 시공ON (Sigong-ON)
- **서비스 개요**: 방수 시공 업계를 위한 통합 SaaS 플랫폼
- **핵심 기능**:
  - AI 기반 누수 기술진단 (사진 분석 → 공법 제안)
  - 자동 견적서 생성 (RAG 기반 적산정보 검색)
  - 프로젝트 관리 (견적~준공까지 전 과정)
  - 노무관리 (일용직 근로계약, 급여, 신고자료)
- **사업모델**: SaaS 구독제 (아이디당 과금, 월/연 선택)
- **호스팅**: 포함

## 2. 계약 전 미팅 (3회)

**장소**: 유니그린 사무실
**참석자**: IT7, 유니그린개발

#### 2.1.1 핵심 요구사항 및 기술 검토

| 기능 | 요구사항 | 기술적 검토 | 결론 |
|------|----------|-------------|------|
| **AI 사진 분석** | 현장 사진 → AI 분석 → 공법 추천 | ChatGPT/Gemini Vision API 활용 가능 | ✅ 실현 가능 (목표 정확도 85%+) |
| **RAG 시스템** | 적산정보 PDF → 자연어 검색 | 한국물가정보 PDF 파싱 → 벡터 임베딩 | ✅ 실현 가능 |
| **견적서 자동화** | 사진 → 기술진단서 → 견적서 → 시방서 → 계약서 | 단계별 데이터 연동 | ✅ 실현 가능 |
| **음성 입력** | 공종 입력 시 음성 지원 | Web Speech API / 모바일 STT | ✅ 실현 가능 |
| **전자계약** | 일용직 근로계약서 전자서명 | 사인 이미지로 법적 효력 | ✅ 실현 가능 (자체 개발) |
| **안전 사진 인식** | 모자/안전화 착용 여부 AI 인식 | 객체 탐지 모델 활용 | ✅ 실현 가능 |

#### 2.1.3 타겟 사용자 및 UX 방향
- **주 사용자**: 40~50대 건설업 종사자
- **UX 원칙**: 최대한 쉽고 직관적인 UI
- **리포트**: 위클리 리포트 제공

#### 2.1.4 호스팅 및 운영

| 항목 | Gemini | ChatGPT |
|------|--------|---------|
| **이미지 판독 정확도** | ⭐⭐⭐⭐⭐ 매우 높음 | ⭐⭐⭐ 보통 |
| **한국 데이터** | 풍부 | 상대적으로 부족 |
| **결론** | ✅ **이미지 판독 우선 모델** | ℹ️ 텍스트 보강/대안 모델 검토 |

**최종 결정**: **이미지 판독은 Gemini 3 계열 우선 사용 (개발 시점 최신 안정 버전 기준)**

#### 2.2.2 시스템 구성도

```
사진 촬영
  ↓
[AI 모듈] 사진 판독 + 공법 제안
  ↓
[RAG 시스템] 적산정보 PDF 검색
  ↓
[ERP 시스템] 견적서 생성
```

#### 2.2.3 적산정보 파싱 결과
- **성공 확인**: 380페이지 × 2개 PDF 파일 파싱 완료
- **검색 방식**: RAG 기반 자연어 검색

#### 2.2.4 전자계약 방식

| 계약 유형 | 구현 방식 | 비고 |
|-----------|-----------|------|
| **공사도급계약** | 검토 중 | 외부 서비스 연동 고려 |
| **근로계약** | 자체 개발 | 사인 이미지 기반 |

#### 2.2.5 모바일 앱 전략

**우선순위**: **안드로이드 우선 개발**

### 3.2 2차 킥오프 (2026.01.26) - 계정 구조 및 기능 구체화

**주제**: 계정/권한 구조, 적산정보, 일용직 관리 세부사항

#### 3.2.1 결제 시스템 확정

**결제 대행사**: **토스페이먼츠 확정**

**선정 이유**:
- 나이스 프로덕션 오류 빈번
- 토스페이먼츠 안정성 높음

#### 3.2.2 중복가입 제한

| 제한 기준 | 대상 | 비고 |
|-----------|------|------|
| **전화번호** | 모든 사용자 | 동일 전화번호 중복 불가 |
| **사업자등록번호** | 고객사, 협력사 | 동일 사업자 중복 불가 |

#### 3.2.3 비밀번호 재설정
**방식**: 휴대전화 OTP 인증

#### 3.2.4 적산정보 관리 상세

**업로드 권한**: 최고관리자(유니그린)만 가능

**처리 방식**:
1. PDF 파일 업로드 (비동기 처리)
2. RAG 시스템 자동 파싱
3. 버전 관리 (이전 버전 보관)
4. 검색 인덱스 재생성

**출처**: 한국물가정보 (연 2회 업데이트)

#### 3.2.5 계정 구조 상세

```
최고관리자 (유니그린개발)
  └── 고객사 대표 (1명, 가입 시 자동 생성)
       ├── 실무자 (이메일 초대)
       └── 현장소장 (이메일 초대)
```

**권한 흐름**:
1. 고객사 가입 시 대표 계정 1개 자동 생성
2. 대표가 실무자를 이메일로 초대
3. 대표가 현장소장을 이메일로 초대

#### 3.2.6 일용직 근로자 가입 (⚠️ 2/2에 변경됨)

**당시 결정**: 노무정보 등록 시 **자동 회원가입**

**프로세스**:

3. 근로자에게 초기 비밀번호 전달
4. 근로자가 앱에서 추가 정보 입력

> ⚠️ **주의**: 이 방식은 2026.02.02 미팅에서 변경되었습니다. [3.3.1](#331-핵심-변경-근로자-가입-방식) 참조

#### 3.2.7 데이터 격리

**원칙**: 고객사 간 근로자 이력 **완전 격리**

**이유**:
- 동일 근로자가 여러 고객사에서 근무 가능

#### 3.3.4 신고자료 Excel 주민번호 처리

**문제**: 주민번호 전체 저장 불가 (개인정보보호법)

**해결 방식**:
1. 생년월일 + 성별 수집
2. 주민번호 앞 7자리 자동 생성: `YYMMDD-G******`
   - YY: 생년 끝 2자리
   - MM: 월
   - DD: 일
   - G: 성별 코드 (1/2: 1900년대생, 3/4: 2000년대생)
3. 뒷자리는 `******`로 마스킹

**예시**:
- 1985년 3월 15일 남성 → `850315-1******`
- 2000년 7월 22일 여성 → `000722-4******`

#### 3.3.5 세금 및 보험료 계산 기준

**일용직 급여 소득세**:

| 일당 범위 | 소득세 | 지방소득세 | 비고 |
|-----------|--------|------------|------|
| **15만원 이하** | 0원 (비과세) | 0원 | - |
| **15만원 초과** | 초과분의 6% | 소득세 × 10% | - |

**4대 보험 공제율** (유니그린 제공 자료 기준):

| 보험 항목 | 요율 | 비고 |

> ※ 보험요율은 정부 정책에 따라 주기적으로 변동되므로, 관리자 화면에서 수정 가능하도록 구현 필요 (유니그린 요청사항)

**계산 예시** (일당 300,000원):

| 공제 항목 | 계산식 | 금액 |
|-----------|--------|------|

#### 3.3.6 동일인 판별 기준

**판별 기준**: **전화번호 + 이름**

**이유**:
- 주민번호 전체 저장 불가

- 이름 추가로 정확도 향상

#### (3) 월별 일용노무비 지급명세서 (통합본)

**요구사항**: 월 단위로 회사 전체의 일용직 근로자 사용 현황을 통합 정리

**용도**: 사업장 세무 신고 기초 자료

**핵심 규칙**:
- 한 월에 여러 현장이 있을 수 있음
- 동일 근로자가 여러 현장에 투입될 수 있음 → **통합하여 표시**

**예시**:
- 근로자 '홍길동'
  - 1/11~1/15: ㅇㅇ중학교 투입
  - 1/16~1/20: ㅇㅇ고등학교 투입
  - → 현장별 명세서: 각 현장에 별도 기입
  - → **통합본**: 1/11~1/20 일용근로 (합산 표시)
