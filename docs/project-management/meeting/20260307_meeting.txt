[확정/합의 사항]

1) 퍼블리싱 진행 현황
- 설계서 화면대로 구현 중이며, 모바일 레이아웃 다듬는 중
- 퍼블리싱 완료 후 IT7에서 URL 공유 예정

2) 자재발주 기능 설계 — 자재마스터 방식 채택
- 자재마스터 메뉴 신설: 플랫폼 관리자(유니그린)가 품목명, 단가, 단위를 사전 등록
- 고객사(시공업체): 자재마스터 목록에서 품목 선택 후 수량만 입력
- 상태 관리 불필요: 발주 후 별도 상태 추적 없음 (1차 범위 제외)
- 발주 데이터는 플랫폼 관리자가 수신하여 조달 처리 (공급사 자동 연동 없음)
- 주요 자재(노루 단일 공급사 기준)만 등록, 소모품/소액 자재는 제외

3) 프로젝트 단계 순서 확정
- 최종 확정 순서: 계약 → 현장대리인 → 자재발주 → 착공 → 작업일지 → 준공
- 기존 "보고서(Report)" 탭은 착공(착공 신고)으로 명칭 변경
- 각 단계가 명확한 공사 진행 흐름을 따르도록 구성

4) 회원가입/구독 프로세스 변경 — 2단계 분리
- 기술적 제약으로 결제 전 계정 생성이 선행되어야 함
- 1단계: 회원가입 (계정 생성)
- 2단계: 무료체험 시작 (약 1개월) → 체험 종료 후 구독 전환
- 체험 기간은 설정 가능하며, 마케팅 상으로는 "무료로 사용해 보기" 강조
- 체험 종료 시점 안내 및 구독 유도 방식은 추후 논의

5) 노임단가 업데이트 방식 — 엑셀 업로드
- 한국물가정보 사이트(유니그린 계정 보유)에서 데이터 수동 추출
- 관리자가 엑셀 파일로 정리 후 플랫폼에 업로드하는 방식으로 구현
- 갱신 주기: 연 2회 (상반기/하반기) — 반기별 수작업으로 충분
- 유니그린이 물가정보 사이트 계정 정보를 IT7에 공유 예정

6) 물가정보 자재단가 — 1차 제외, 수기 입력으로 대체
- 물가정보 DB(수천 개 품목)는 1차 범위에서 제외
- 현장단가(사용자 직접 입력) 방식으로 대체
- 조달청 공공데이터 API를 통한 자재단가 연동 가능 여부는 IT7이 별도 검토 예정

7) AI 진단 프롬프트 고도화 방향
- 포지티브/네거티브 프롬프트 분리: 진단 결과 품질 향상을 위해 구조화
- 자재 추천 범위 한정: AI가 추천하는 자재를 자재마스터 목록 내 품목으로 제한
- 세부 프롬프트 설계는 추후 고도화 시 논의
