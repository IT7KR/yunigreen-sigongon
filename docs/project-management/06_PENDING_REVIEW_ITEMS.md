## 3차 미팅 (2026.01.08) 미완료 액션아이템

### 개요

계약 전 최종 협의 미팅에서 발생한 액션아이템 중 미완료 사항입니다.

| # | 검토 항목 | 담당 | 상태 | 상세 |
|:--:|----------|:----:|:--:|------|
| 1 | 나라장터 계정 연동 API 확인 | IT7 | 미확인 | 공식 API 미제공 예상. 검색 기반 조회는 가능하나 계정 기반 자동 연동 불가능성 높음. **대응**: 수기 입력 또는 스캔본 업로드로 계약 관리 |
| 2 | 신분증 마스킹 보안법 확인 | IT7 | 미확인 | 일용근로자 신분증 촬영 시 이미지 저장 방식 논의. 주민번호 마스킹 필요 여부 확인 필요. **관련법**: 개인정보보호법 제18조 |
| 3 | 명세서 V2 작성 | IT7 | 미확인 | 1차/2차 분리 견적서 기반 명세서 v2 작성. 추가로 나라장터/관리자 페이지 내용 반영 후 v3까지 조율 예상. **현황**: 계약서 협의 후 최종 확정 예정 |

## 1차 킥오프 미팅 (2026.01.21) 오픈 이슈

### 개요

프로젝트 착수 미팅에서 논의 중인 열린 이슈들입니다. 착수 전 또는 착수 초기에 확인 필요합니다.

| # | 검토 항목 | 담당 | 상태 | 상세 |
|:--:|----------|:----:|:--:|------|
| 4 | 사업자등록번호 검증 API 비용/제약 | IT7 | 미확인 | 고객사 가입 시 사업자등록번호 자동 검증 기능. **추정 API 후보**: 국세청 OPEN API, 더존 비즈넷. **확인 필요**: 비용, 요청 제한, 실명인증 필요 여부 |
| 5 | 여성기업 여부 확인 API | IT7 | 미확인 | 공공 발주 시 여성기업 여부 확인 필요. **추정 API 후보**: 중소벤처기업부 API. **확인 필요**: 제공 여부, 접근성, 실시간 검증 가능성 |
| 6 | 한글(HWP) 자동 생성 POC | IT7 | 미확인 | 시방서/계약서 등을 한글 파일로 직접 생성할 수 있는지 기술 검증. **현황**: hwpx 표준화 미흡. **대응 방향**: 기술 검증 후 PDF/DOCX 우선 → 한글은 템플릿으로 대체 검토 |
| 7 | 문서 출력 형식 최종 결정 | IT7 | 부분 확정 | **결정 사항**: Excel(견적서), PDF(계약서), DOCX(사진첩), PDF/DOCX(시방서 등). **미결정**: 한글 원본 제공 가능성, XLSX vs XLS 선택 |

| 11 | 요금 정책 상세 | IT7 | 미확인 | SaaS 구독형 요금제 상세 결정. **확정 사항**: 아이디당 과금, 월/연 선택. **미결정**: 단계별 가격표, 추가 기능 유료화 범위, 무료 체험 기간 연장 여부 |
| 12 | 노무관리 화면 검수 | IT7 | 미확인 | 스토리보드 설계 단계에서 노무비 신고, 일용근로자 관리 화면 설계 검수. **참고**: 유니그린 기존 시스템의 노무관리 프로세스 확인 필요 |

IT7 내부에서 기술 검토 후 의사결정이 필요한 항목들입니다.

| # | 검토 항목 | 담당 | 상태 | 상세 |
|:--:|----------|:----:|:--:|------|
| 13 | HWP→PDF 변환 방식 | IT7 | 미확인 | 한글 템플릿 파일을 웹에서 PDF로 변환하는 기술 방식. **옵션**: 라이브러리(LibreOffice 활용), API 서비스(아래아한글 API), 자체 변환 엔진. **권장**: 신뢰도 높은 API 서비스 사용 |
| 14 | Excel 생성 라이브러리 역량 | IT7 | 미확인 | 견적내역서, 공사일지 등 Excel 파일 자동 생성 라이브러리 선택. **기술 스택**: Node.js (ExcelJS, node-xlsx) 또는 Python (openpyxl, pandas). **확인 필요**: 복잡한 수식/포맷 지원 가능성 |
| 15 | 교육청 원클릭 프로그램 (.xlsm) 대응 | IT7 | 미확인 | 학교 발주 프로젝트용 교육청 제출 Excel 매크로 파일 생성. **현황**: 대부분 .xlsm 또는 .xls 매크로 포함. **확인 필요**: 웹에서 동작 가능한 .xlsm 생성 방법, 매크로 변경/보존 가능성 |
| 16 | 폐기물 증빙서류 관리 방식 | IT7 | 미확인 | 폐기물 처리 증명서 관리 및 추적. **옵션**: 사용자 업로드만, 또는 폐기물 처리업체 API 연동. **권장**: 1차에는 업로드 방식으로 진행, 향후 확장 |
| 17 | 전문건설공제조합 보증서 발급 연동 | IT7 | 미확인 | 계약보증서 자동 발급 또는 조회 API. **현황**: 전문건설공제조합 공식 API 없음. **대응 방안**: 보증서 PDF 업로드 + 수기 입력. 장기 목표: 보증서 조회 스크래핑 기술 검토 (정보보안 위배 위험 있음) |

## 문제별 해결 방안

### 외부 API 미제공 (나라장터, 공제조합)

**현황**: 공식 API가 제공되지 않는 경우

**대응 방안**:
1. **1차**: 수기 입력 또는 스캔본/PDF 업로드
2. **2차**: 사용자 편의성 향상 (매뉴얼 도입)
3. **향후**: 웹 크롤링 또는 스크래핑 검토 (정보보안 위배 확인 필수)

### 한글 파일 생성의 기술적 제약

**현황**: 웹에서 .hwp 파일 생성 기술적 한계

**대응 방안**:
1. **현재**: PDF + DOCX 제공 (파워유저용)
2. **선택**: 한글 템플릿 파일 다운로드 후 수기 입력 방식
3. **향후**: hwpx 표준화 진전 시 재검토

### 보안/개인정보 관련 이슈

**현황**: 신분증 마스킹, 개인정보 동의 등 법적 검토 필요

**대응 방안**:
1. **즉시 확인**: 개인정보보호법 제18조, 신용정보보호법 검토
2. **구현**: 주민번호 마스킹, 접근권한 제한, 감사 로그 기록
3. **보안**: 암호화 저장, 만료 시 자동 삭제 프로세스
