# 유니그린 SaaS - 디자인 시스템 가이드

## 1. 디자인 철학

### 1.1 핵심 원칙: "현장 우선, 신뢰성"

우리의 사용자는 도전적인 환경에서 일하는 건설 기술자입니다:
- **야외 환경**: 강한 햇빛, 먼지, 비
- **한 손 조작**: 도구를 들고 있거나 사다리 위에서 작업
- **시간 압박**: 작업 사이사이 빠른 데이터 입력
- **비전문가**: 간단하고 명확한 인터페이스 필요

### 1.2 디자인 우선순위

1. **가독성**: 높은 대비, 큰 글씨
2. **터치 친화적**: 최소 48px 터치 타겟
3. **속도**: 최소한의 탭으로 작업 완료
4. **오프라인 인식**: 명확한 동기화 상태 표시
5. **전문성**: B2B 신뢰감 있는 미학


```
│  PRIMARY        │  틸 채움, 흰색 텍스트                      │
```

## 9. 접근성 (a11y)

### 9.1 색상 대비

- 일반 텍스트: 최소 4.5:1
- 큰 텍스트 (18px+): 최소 3:1
- 인터랙티브 요소: 최소 3:1

### 9.2 포커스 상태

모든 인터랙티브 요소는 가시적인 포커스가 있어야 합니다:

```css
:focus-visible {
  outline: 2px solid var(--color-primary-500);
  outline-offset: 2px;
}
```

### 9.3 스크린 리더 지원

- 모든 이미지: `alt` 텍스트
- 아이콘만 있는 버튼: `aria-label`
- 폼 필드: 연관된 `<label>`
- 동적 콘텐츠: `aria-live` 영역
