> **근거 문서**: 계약 전 미팅 (2025.12.10, 12.18, 2026.01.08), 킥오프 미팅 (2026.01.21)

## 1. 문서 개요

### 1.1 목적
본 문서는 시공ON 플랫폼의 주요 사용자 흐름(User Flow)을 정의합니다. 각 흐름은 스토리보드 작성과 화면 명세서의 기초 자료로 활용됩니다.

### 1.2 대상 독자
- UI/UX 디자이너
- 프론트엔드 개발자
- 프로젝트 매니저
- QA 엔지니어

### 1.3 사용자 역할 정의

| 역할 | 설명 | 주요 플랫폼 |
|------|------|-------------|
| 슈퍼 어드민 | 유니그린 관리자, 전체 시스템 관리 | 웹 (관리자) |
| 고객사 대표 | 업체 대표/관리자, 전체 업무 접근 | 웹 + 앱 |
| 현장 소장 | 현장 관리자, 착공~준공 담당 | 앱 (주) + 웹 (보조) |
| 일용직 근로자 | 일용 근로자, 계약/지급 확인 | 앱 |

## 2. 고객사 등록/온보딩 흐름

### 2.1 흐름 개요

```
서비스 접속 → 회원가입 → 사업자 인증 → 요금제 선택 → 결제 → 초기 설정 → 대시보드
```

### 2.3 분기 조건

| 조건 | 분기 |
|------|------|
| 사업자번호 검증 실패 | 오류 메시지 → 재입력 요청 |
| 무료 체험 선택 | 결제 정보 선입력 → 바로 대시보드 |
| 앱에서 결제 시도 | 웹 결제 페이지로 안내 |

### 2.4 관련 화면 목록

- `SCRN-001`: 랜딩 페이지
- `SCRN-002`: 회원가입 폼
- `SCRN-003`: SMS 인증 모달
- `SCRN-004`: 사업자 인증 폼
- `SCRN-005`: 요금제 선택 페이지
- `SCRN-006`: 결제 페이지 (토스페이먼츠)
- `SCRN-007`: 초기 설정 마법사
- `SCRN-008`: 대시보드

```
프로젝트 생성 → 현장방문 → AI 진단 → 견적서 → 계약 → 착공 → 시공 → 준공 → 정산
```
